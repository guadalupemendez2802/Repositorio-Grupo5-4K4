## ConvenciГіn de Nombrado de Commits

Los mensajes de commit deben seguir el siguiente formato: `<prefijo>: descripciГіn breve en espaГұol`

| Prefijo   | Contexto                                        |
|-----------|------------------------------------------------|
| `feat`    | Nueva funcionalidad o caracterГӯstica            |
| `fix`     | CorrecciГіn de un error o bug                    |
| `docs`    | Cambios o mejoras en la documentaciГіn           |
| `refactor`| Mejora en el cГіdigo sin cambiar su funcionalidad|

**Reglas adicionales:**
- La descripciГіn en **minГәsculas**, sin punto al final, con el verbo en **presente**
- El mensaje siempre en **espaГұol**
- Si hay dudas entre `feat` y `docs`: si es contenido nuevo вҶ’ `feat`; si es una mejora de algo existente вҶ’ `docs`

Para mГЎs detalles, consultar [agente_push/instrucciones.md](agente_push/instrucciones.md).
